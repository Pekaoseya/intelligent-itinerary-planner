# 反馈功能使用说明

## 功能概述

智能助手新增了反馈功能，用户可以方便地提交 Bug 报告、功能建议或其他类型的反馈。系统会自动包含当前对话内容，帮助开发者快速定位问题。

## 功能特点

### 1. 自动保存对话内容
- 提交反馈时，系统会自动保存当前会话的所有对话消息
- 帮助开发者复现问题和理解上下文

### 2. 多种反馈类型
- **Bug** 🐛：报告功能缺陷或错误
- **建议** 💡：提出新功能或改进建议
- **其他** 📝：其他类型反馈

### 3. 用户信息记录
- 自动记录用户位置信息（如果已授权）
- 自动记录提交时间

## 使用方法

### 在聊天界面中

1. **打开反馈弹窗**
   - 在聊天界面右上角点击"反馈"按钮（🐛 图标）

2. **选择反馈类型**
   - 点击对应的按钮选择反馈类型

3. **描述问题**
   - 在文本框中详细描述问题或建议
   - 系统会自动包含当前对话内容（会显示消息数量）

4. **提交反馈**
   - 点击"提交"按钮提交反馈
   - 提交成功后弹窗会自动关闭

## 技术实现

### 后端实现

**文件结构：**
```
server/src/modules/feedback/
├── feedback.controller.ts  # 控制器
├── feedback.service.ts      # 服务
└── feedback.module.ts       # 模块
```

**API 接口：**
- `POST /api/feedback/submit` - 提交反馈

**数据存储：**
- 反馈内容以 JSON 文件形式保存在 `server/feedbacks/` 目录
- 文件命名格式：`{timestamp}-{userId}.json`

**反馈数据结构：**
```json
{
  "id": "1776132814289-test-use",
  "type": "bug",
  "userId": "test-user",
  "description": "测试反馈功能",
  "messages": [
    {
      "role": "user",
      "content": "测试消息"
    }
  ],
  "userInfo": {
    "location": {...},
    "timestamp": "2026-04-14T02:13:34.289Z"
  },
  "timestamp": "2026-04-14T02:13:34.289Z"
}
```

### 前端实现

**组件结构：**
```
src/components/feedback/
├── feedback-modal.tsx    # 反馈弹窗组件
├── feedback-button.tsx   # 反馈按钮组件
└── index.ts             # 导出
```

**服务层：**
```
src/services/feedback.service.ts  # 反馈服务
```

**页面集成：**
- 集成到 `src/pages/index/index.tsx`
- 在定位栏右侧添加反馈按钮
- 支持自动传递当前对话内容和用户信息

## 开发者查看反馈

### 查看所有反馈

反馈文件保存在 `server/feedbacks/` 目录，可以手动查看或编写脚本批量分析。

### 示例：查看最新反馈

```bash
# 查看最新的反馈文件
ls -lt server/feedbacks/ | head -5

# 查看反馈内容
cat server/feedbacks/1776132814289-test-use.json
```

### 反馈分析建议

1. **按类型分类**
   - Bug：需要优先修复
   - 建议：纳入产品规划
   - 其他：根据具体情况处理

2. **复现问题**
   - 使用反馈中的对话内容复现问题
   - 查看用户位置和时间信息了解上下文

3. **跟进反馈**
   - 修复后告知用户
   - 记录处理结果

## 注意事项

1. **隐私保护**
   - 反馈内容仅用于问题排查
   - 不要在反馈中包含敏感信息

2. **反馈质量**
   - 详细描述问题和复现步骤
   - 系统自动包含对话内容，无需重复粘贴

3. **网络要求**
   - 提交反馈需要网络连接
   - 提交失败会显示错误信息
